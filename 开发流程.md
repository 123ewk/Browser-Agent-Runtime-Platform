# 二、开发流程（推荐顺序）

&#x20;

### 阶段 0：基础准备

- 初始化项目仓库（前端/后端/agent）
- 配置 Playwright 环境
- 搭建 PostgreSQL + Redis + MinIO
- 创建虚拟环境 / 安装依赖

***

### 阶段 1：Browser Agent 核心

- 实现 DOM-first 浏览器操作 Skill
  - 打开网页
  - 登录模拟
  - 点击/填表/上传
- Browser Context Persistence
  - 保存 / 恢复 storage\_state
- 简单 Task Runtime
  - 单任务执行
  - 日志输出

***

### 阶段 2：边思考边执行 Runtime

- Planner + Executor
- 状态机实现（RUNNING / PAUSED / WAITING\_USER / COMPLETED / FAILED）
- Skill 调用机制
- Checkpoint 保存

***

### 阶段 3：Human-in-the-loop

- Semi-auto Mode
- YOLO Mode
- 中途插话 / interrupt
- Task resume / replanning

***

### 阶段 4：Observability

- Timeline / Skill Trace / Screenshot Timeline
- Token / Cost 统计
- 前端 WebSocket 显示
- 操作回放

***

### 阶段 5：多任务并发

- 多任务执行 / Scheduler
- Browser Context 复用策略
- 并发日志 & checkpoint管理

***

### 阶段 6：Skill 扩展

- Skill Metadata + Discovery
- Skill 上传接口
- Skill 内部微代理化
- 权限管理

***

### 阶段 7：数据分析 & 文档生成 Skill

- 数据分析 Skill (pandas)
- PDF / Excel 生成 Skill
- 内容生成 Skill
- 实际任务 Demo

***

### 阶段 8：整合 & 测试

- 前端展示
- 后端 API 完整性
- Agent Runtime 全流程测试
- 异常恢复测试
- 并发任务测试

***

### 阶段 9：优化 & 高级特性

- Vision fallback / selector 自修复
- Token / Cost 优化
- Replay & 回放功能完善
- UI 优化 & 用户体验
