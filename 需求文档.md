# 项目定位（核心定义）

## 项目名称（暂定）

# Browser Agent Runtime Platform

一个：

# 面向运营场景的 AI Browser Agent 平台

用户只需要输入目标：

```
“帮我发布今天的抖音内容”
“帮我分析销售数据并生成 PDF”
“帮我录入后台数据”

```

Agent 即可：

- 自动规划
- 自动执行
- 自动操作网页
- 自动观测页面
- 自动恢复失败步骤
- 支持人工确认
- 支持用户中途插话
- 支持 checkpoint 恢复
- 支持多任务并发

***

# 项目核心卖点（面试核心标签）

# Browser Agent Runtime

重点不是：

- AI 聊天
- RAG
- 普通 workflow

而是：

# “AI 自动操作真实网页的软件执行 Runtime”

这是项目真正的差异化。

***

# 项目核心能力

***

# 1. Browser Agent（核心）

## DOM-first Browser Agent

使用：

- Playwright
- Browser Context Persistence

实现：

- 自动登录
- 自动点击
- 自动填表
- 自动发布内容
- 自动上传文件
- 自动操作后台

***

## Vision Fallback（辅助）

仅在：

- selector 失效
- 页面变化
- DOM 无法定位

时启用：

```
截图
→ 多模态模型分析
→ 找新元素
→ 自动修复 selector
→ retry

```

实现：

# Self-healing Browser Agent

***

# 2. Skill-based Runtime

Agent 不直接拥有所有 Tool。

而是：

# Skill 动态能力系统

***

## Skill 本质

Skill =

```
Prompt
+
Executable Code
+
Metadata
+
Permissions

```

***

## Skill 示例

### browser\_operator\_skill

负责：

- Playwright 操作
- 页面观测
- DOM 分析

***

### data\_analysis\_skill

负责：

- pandas 分析
- 图表生成
- CSV 处理

***

### pdf\_generator\_skill

负责：

- PDF 生成
- 报告导出

***

### content\_generation\_skill

负责：

- 文案生成
- SEO 标题
- 电商内容

***

# 3. 边思考边执行（核心架构）

Agent：

# ReAct Runtime

不是：

```
先生成完整计划
再执行

```

而是：

```
观察
→ 思考
→ 执行一步
→ 再观察
→ 动态调整

```

原因：

- 网页是动态的
- 页面状态不可预测
- 支持实时恢复
- 支持用户中断
- 支持动态 replanning

***

***

# Runtime 核心架构

***

# Runtime 类型

## Workflow Agent

不是：

Persistent Agent。

当前阶段：

# 可恢复 Workflow Runtime

***

# Runtime 模式

## 单 Agent Runtime

但：

# Skill 内部允许微代理化

例如：

```
browser_skill

```

内部：

- selector 修复子 agent
- 页面分析子 agent

但：

系统外部只有：

# 一个主 Agent Runtime

***

# Runtime State Machine（核心）

系统采用：

# 事件驱动状态机

***

## 核心状态

```
PENDING
PLANNING
RUNNING
WAITING_USER
PAUSED
RETRYING
RECOVERING
COMPLETED
FAILED
CANCELLED

```

***

## Runtime Event

```
USER_INTERRUPT
USER_MESSAGE
STOP
RESUME
CHECKPOINT_RESTORE
RETRY

```

***

# Browser Runtime 架构

***

# Browser Runtime

```
Browser
  ↓
Browser Context
  ↓
Page

```

***

# Browser Context Persistence

系统保存：

```
context.storage_state()

```

恢复：

```
browser.new_context(storage_state=...)

```

***

# Browser Session Strategy

## 同一任务：

复用同一 Browser Context

***

## 不同任务：

隔离不同 Browser Context
