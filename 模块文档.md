# 一、模块拆分

&#x20;

## 1️⃣ 前端模块（Next.js + React + Tailwind）

### 1.1 Agent 控制台

- **任务列表**
  - 显示所有任务及状态（PENDING / RUNNING / WAITING\_USER / COMPLETED / FAILED）
  - 支持搜索和筛选
- **任务详情**
  - Timeline 流
  - Screenshot 流
  - Skill Trace
  - Cost / Token 使用
- **操作按钮**
  - 中断 / resume / cancel / retry
- **任务并行管理**
  - 多任务同时显示状态

### 1.2 人机交互窗口

- 用户确认危险操作（Semi-auto Mode）
- 用户插话（修改任务/参数）
- YOLO 模式切换
- 插件式 Skill 调用界面（可以上传 skill.md 或 JSON 配置）

### 1.3 Browser Replay & Observability

- 操作回放
- 每步操作截图时间线
- 日志回溯
- 可视化状态机展示

***

## 2️⃣ 后端模块（FastAPI）

### 2.1 Runtime API

- 接收前端任务请求
- 任务调度
- 提供事件流（WebSocket）给前端
- 查询历史任务、成本统计、日志、截屏

### 2.2 Skill Management

- Skill 元数据存储（名称、描述、权限、执行入口）
- Skill 加载/卸载
- Skill 运行上下文
- Skill 调用接口（Executor）

### 2.3 Task / Agent Runtime

- 状态机管理
- Checkpoint 保存与恢复
- 异常处理与 retry
- Human-in-the-loop 事件处理
- 并发任务调度

### 2.4 Browser Session Management

- Browser Context 持久化 / 恢复
- Session 隔离
- Session 与任务绑定策略
- 浏览器状态快照 storage\_state.json

***

## 3️⃣ Agent 核心模块（LangGraph + Playwright + LLM）

### 3.1 Planner

- 根据用户目标动态生成下一步执行动作
- 根据 Skill 描述动态选择 Skill
- 支持边思考边执行
- 中途可被 Human-in-the-loop 打断重新规划

### 3.2 Executor

- 调用对应 Skill 执行动作
- DOM-first 浏览器操作
- Vision 辅助 fallback
- Capture Screenshot / Log / State

### 3.3 Recovery / Checkpoint

- 保存执行状态
- 崩溃或异常恢复
- selector 失效自动重试
- 支持恢复到上一步或上个 checkpoint

### 3.4 Observability

- Timeline / Log / Screenshot / Skill Trace
- Token / Cost Tracking
- Replay 支持
- 前端显示接口

***

## 4️⃣ 数据与存储模块

### 4.1 数据库（PostgreSQL + pgvector）

- Task 信息：ID / 状态 / StartTime / EndTime
- Checkpoint / Agent 状态
- Skill 元数据
- Task Embedding / Memory（未来可扩展 RAG）

### 4.2 缓存 / 消息（Redis）

- Task Event Queue
- Pub/Sub 实时推送前端
- 并发任务调度

### 4.3 对象存储（S3 / MinIO）

- Screenshot / Replay / PDF / 文件导出

***

## 5️⃣ AI 模型模块

### 5.1 LLM

- 任务规划
- Skill 调度
- Prompt 模板管理
- 边思考边执行逻辑

### 5.2 Vision / VLM

- 页面元素识别
- Selector fallback
- 页面截图分析

### 5.3 本地或云模型

- DeepSeek / MiMo / MiniMax
- 可切换不同 provider
